Introduction

Assembly

Assembly is a unique language. It is also unlike any other language in that it is an untyped language and it has direct access to the CPUs internals. It gives the programmer control over everything, but the programmer must "think like the computer" in a manner of speaking.

No matter what processor you have, when it is turned on it will start at a specific address and start to look for a command to execute. If there is no firmware the address pins be set to the address to the start address and then it will read in data on the data pins, looking for a command to execute.

In the case of an x86 computer there will be a BIOS (and/or UEFI). We will look into BIOS programming in this section.
