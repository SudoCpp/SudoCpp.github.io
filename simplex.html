Simplex Toolkit

The Simplex Toolkit is a set of C++ classes with the goals to make C++ development as easy as possible. Often the approach with C++ projects has become "write the parts that need to be fast in C/C++ and write the rest in something else". That approach does work, but how do you mesh the two together? COM? While there are multiple solutions. The approach for the Simplex Toolkit is write everything in C++ in a simle fashion and then if something is extermely speed crtical, just write that part the way you desire.

Goals

  • Use a very object oriented approach.
  • Make development as fast and easy as possible.
  • Be lightweight.
  • Automate memory management through RAII.

Usage Patterns

  • Namespaces are all lowercase.
  • Member functions/methods are camelCase (starting with lowercase)
  • Classes are PascalCase (starting with uppercase)
  • If a function calls for a pointer (*) it is taking over ownership. (You no longer need to free/delete it)
  • If a function calls for a reference (&) ownership is not transferred.
